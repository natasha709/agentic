# Security Incident Response Guide

## Overview
This guide outlines procedures for handling common security incidents.

## Incident Categories

### 1. Compromised Account
**Signs:**
- Unusual login locations/times
- Multiple failed login attempts
- Password reset requests you didn't make

**Immediate Actions:**
1. Disable the account immediately
2. Force logout all sessions
3. Check recent activity logs
4. Reset password
5. Enable MFA if not already active

### 2. Malware Detection
**Signs:**
- Unexpected popups
- Slow computer performance
- Unknown processes running
- Browser redirects

**Immediate Actions:**
1. Disconnect from network
2. Run full antivirus scan
3. Check for suspicious processes
4. Review startup programs
5. Restore from clean backup if needed

### 3. Phishing Attempt
**Signs:**
- Suspicious emails requesting credentials
- Unexpected attachments
- Urgent action requests

**Actions:**
1. Do NOT click any links
2. Report to IT security
3. Delete the email
4. If clicked: change passwords immediately

### 4. Data Breach
**Signs:**
- Unusual data access patterns
- Large data exports
- Unauthorized access alerts

**Actions:**
1. Isolate affected systems
2. Document the incident
3. Notify security team
4. Assess data exposure
5. Report to management

## Security Best Practices

- Use strong, unique passwords
- Enable multi-factor authentication
- Keep software updated
- Don't install unauthorized software
- Report suspicious activity immediately
